# 智能知识保存功能使用指南

## ✅ 功能已启用

工作流已增加**智能知识保存节点**，会自动将高价值内容保存到知识库！

---

## 🎯 工作流程

```
搜索 → 知识库检索 → 分析 → 推送 → 智能保存
                                    ↓
                              自动判断价值
                                    ↓
                          ┌─────────┴─────────┐
                          ↓                   ↓
                      值得保存           不值得保存
                          ↓                   ↓
                    检查相似度            跳过保存
                          ↓
                   ┌──────┴──────┐
                   ↓             ↓
                无重复        已有相似内容
                   ↓             ↓
               自动保存       跳过保存
```

---

## 📊 智能判断标准

### ✅ 自动保存的内容

工作流会自动保存以下类型的高价值内容：

| 类型 | 说明 | 判断依据 |
|------|------|---------|
| **行业趋势报告** | 长期参考价值的趋势分析 | 数据支撑、趋势预测 |
| **技术案例分析** | 可复用的技术方案 | 创新点、可行性分析 |
| **用户需求洞察** | 深度用户研究结论 | 用户数据、行为分析 |
| **项目经验总结** | 有借鉴意义的经验 | 成功/失败教训、优化建议 |
| **竞品分析报告** | 深度对比分析 | 多维度对比、优劣势分析 |

### ❌ 自动跳过的内容

工作流会自动跳过以下低价值内容：

| 类型 | 说明 | 判断依据 |
|------|------|---------|
| **日常新闻资讯** | 时效性短，几天后过时 | 缺乏长期价值 |
| **重复或相似内容** | 知识库已有相似知识 | 相似度 > 0.85 |
| **低质量信息** | 缺乏数据支撑 | 未验证、主观性强 |
| **无关内容** | 超出专业领域 | 与座椅无关 |

## 🔧 技术实现

### 步骤1: AI价值评估

使用LLM判断内容的知识价值：

```python
# 评估维度
{
    "should_save": true/false,      # 是否值得保存
    "reason": "判断理由",            # 判断依据
    "value_type": "知识类型",        # 内容分类
    "confidence": 0.0-1.0           # 评估置信度
}
```

### 步骤2: 相似度检测

检查知识库是否已有相似内容：

```python
# 检索知识库
search_response = knowledge_client.search(
    query=content[:200],
    table_names=["Car_Seat"],
    top_k=3,
    min_score=0.85  # 相似度阈值
)

# 如果相似度 >= 0.85，跳过保存
```

### 步骤3: 自动保存

将高价值内容添加到知识库：

```python
# 构建知识内容（含元数据）
knowledge_content = f"""【{value_type}】汽车座椅产品规划日报 - {date}

{analysis_content}

---
知识类型: {value_type}
评估置信度: {confidence}
保存时间: {date}
"""

# 添加到知识库
knowledge_client.add_documents(
    documents=[...],
    table_name="Car_Seat"
)
```

## ⚙️ 配置参数

### 相似度阈值

控制重复检测的严格程度：

```python
# 在 src/graphs/nodes/save_knowledge_node.py 中
min_score=0.85  # 相似度阈值（0.0-1.0）

# 调整建议：
# - 0.90: 非常严格，只避免高度重复
# - 0.85: 平衡，推荐值
# - 0.80: 宽松，允许相似内容
```

### 置信度阈值

虽然代码中没有显式设置，但可以在判断时增加：

```python
# 只保存置信度 > 0.7 的内容
if confidence < 0.7:
    return SaveKnowledgeNodeOutput(
        save_status="跳过保存：评估置信度过低",
        saved_content=""
    )
```

## 🔍 故障排查

### 问题1: 没有保存任何内容

**可能原因：**
- 分析内容不符合保存标准
- 相似度检测误判
- 知识库权限问题

**解决方案：**
1. 检查分析结果质量
2. 调整相似度阈值
3. 查看错误日志

### 问题2: 保存了低质量内容

**可能原因：**
- AI判断标准过宽
- 置信度阈值过低

**解决方案：**
```python
# 提高置信度要求
if confidence < 0.75:  # 从默认提高到0.75
    return "跳过保存"
```

### 问题3: 重复保存相似内容

**可能原因：**
- 相似度阈值过低
- 检索失败

**解决方案：**
```python
# 提高相似度阈值
min_score=0.90  # 从0.85提高到0.90
```
