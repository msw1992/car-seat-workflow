# 搜索优化方案说明

## 背景

用户要求添加站点白名单，仅搜索指定的权威汽车资讯网站。

## 测试结果

### 站点限制测试

**测试命令：**
```bash
coze-coding-ai search "汽车座椅 产品规划" --sites "autohome.com.cn,d1ev.com" --time-range "1w" --count 5
```

**结果：** ❌ 没有找到搜索结果

**不带站点限制测试：**
```bash
coze-coding-ai search "汽车座椅 产品规划" --time-range "1w" --count 5
```

**结果：** ✅ 找到 5 条结果

## 问题分析

1. **站点限制过于严格**：在指定站点内搜索"汽车座椅 产品规划"且时间范围为"最近一周"，可能确实没有匹配内容
2. **内容时效性**：权威站点不一定每天都有座椅相关内容发布
3. **搜索精度**：站点限制+时间限制+关键词，三重过滤可能导致结果为空

## 推荐方案

### 方案A：质量后评估（当前采用）

**策略：**
- 搜索时不限制站点，保证能获取足够的结果
- 在分析节点中，让LLM根据内容质量和站点权威性进行评估
- 在保存节点中，优先保存来自权威站点的内容

**优点：**
- ✅ 保证搜索结果数量
- ✅ 不遗漏有价值的内容
- ✅ 通过AI评估保证质量

**实现：**
已在 `analysis_node` 中添加站点质量评估逻辑

### 方案B：双重搜索策略

**策略：**
- 第一次搜索：仅权威站点，如果结果<10条
- 第二次搜索：全网搜索，补充结果

**优点：**
- ✅ 优先获取权威内容
- ✅ 保证结果数量

**缺点：**
- ❌ 需要两次API调用
- ❌ 实现复杂度增加

### 方案C：等待SDK改进

**策略：**
- 暂时不使用站点限制
- 等待SDK提供更灵活的站点过滤功能
- 或等待指定站点有更多相关内容
